    Http Header Referer

    2007-03-23
    Life

    It seems that Pentaho Pre-Configged server doesn’t support reporting security.That means everyone can see the report if he know the URL.
    And I found out I could get the users’ role set in JBoss portal session but not Pentaho session since I used two filter in both session.
    Then I think they are different sessions and I tried to find the relationship but failed.
    So I didnot know how to transfer the role infor from JBoss portal session to Pentaho session.Our PM gave me another solution using the http header referer attribute. the method is:

    String referer = (String)request.getHeader().getAttribute(“referer”);

    For now I just validated whether the value is NULL.Because if someone want to get report with direct URL accessing,the value is NULL. So:

    if (referer != null){
    pentahoSession.setAttribute(“projectfilter”,new Object());
    }else{
    pentahoSession.setAttribute(“projectfilter”,“empty”);
    }
    It seems very confused that XAction cannot get the value if it’s just a string!
    And it works.We were a little excited!
