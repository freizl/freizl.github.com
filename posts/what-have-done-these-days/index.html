What have done these days

2007-04-21
Life

It seems that BIRT is a easy to learn and great reporting tool. I designed several reports with it.
Not very difficult and I also used its shared libraries feature, very cool. But we met a hard time of the drill through report, which also used shared libraries.
Everything goes well of each single report in BIRT Designer but failed when combining them through drill down,either deploying to Pentaho server.
I totally had no idea of how to do with this issue at first. Then I got FileMon (designed by Sysinternal) to find out where the Pentaho try to get the libraries files and images.
Sounds interesting, the directory is {pentaho-demo/jboss/bin}. Well, Pentaho could show the report after I place necessary files into that directory.
But our PM said place file into the jboss/bin is not a good idea. So, we need to keep researching.

We configure LDAP login module for authentication In the Pentaho GA-1.2.0. But now there must be local user whose name should be identical with LDAP user name.
It said in the Pentaho later release, there are several extended login module can auto synchronize local user with remote server.
Sounds great. Wish I got time to test it.

I also got start to use Ant in project. I found it’s a so good tool that every web application project should include it!
The process of deployment turn out to be very easy as long as project structure doesn’t change a lot.
It seems that core tasks of Ant are a little limited. so how to pull into more tasks?
Maybe I also need to study Maven 2 because it’s very popular these days after all.
